使用docker+nodejs+gradle搭建android持续集成环境

作者 JsonLu 日期 2017-09-22
使用docker+nodejs+gradle搭建android持续集成环境

如题，需要搭建一套满足开箱即用的持续集成打包环境（笔者太懒，不喜欢给别人打包），满足非技术开发的日常工作之需。

以下是环境构成，介于打包环境搭建比较复杂，笔者就做了一个docker镜像，免去搭建环境或者环境因素导致的各种不愉快；这里不再赘述docker镜像如何制作如何使用等等过程，需要的童鞋自行百度。

接下来需要项目支持gradle命令打包并上传到第三方存储位置

项目构建脚本支持

需要在build.gradle 中支持打包并上传脚本，增加task操作

在gradle.properties中增加几个全局配置file1、file、_api_key、uKey配置
file、file1为不同版本的打包文件位置，填写的时候注意路径
_api_key、uKey为蒲公英的配置信息

nodejs应用支持

nodejs应用作为前置接受数据并转换数据的应用，负责调起git、gradle操作命令，操作成功后推送消息到钉钉的工作，通过docker数据卷运行在docker容器中。

源码在我的github项目链接，需要配置的内容主要在config.json文件中。
data.group节点里面是团队下面配置的钉钉token地址

data.git下面是一些项目的git地址、分支信息、团队信息（关联data.group下面的数据）

请求url采用rest接口方式 ip:8086/build/:group/:project/:branch/:task，task需要注意代码中的配置

例如 ip:8086/api/build/android-developers/StaffLoan/dev/1
